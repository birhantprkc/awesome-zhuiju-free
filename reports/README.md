# 验证与投诉记录

这里保存资源的历史记录，确保评分与下架决定可以被追溯。

## 文件

- `verifications.json`：每次人工或自动验证的结果，只追加、不覆盖历史。
- `notices.json`：权利通知、安全投诉和其他正式处理记录。

## 记录原则

- 不记录与处理无关的个人信息。
- 未核实的投诉状态使用 `received`，不要直接写成事实。
- 资源被降级、暂停或下架时，应写明原因和关联记录。
- 投诉撤回或处理完成后保留历史，但更新处理状态。

投诉状态：

- `received`：已收到，等待复核
- `reviewing`：正在复核
- `actioned`：已采取措施
- `rejected`：材料不足或不适用
- `withdrawn`：提交方撤回
